**Status**: Active
**Effective Date**: 2025-12-27
**Version**: 1.0 (Initial)
**Owner**: CTX-00 (Context Manager)
**Applies To**: All 17 agents in the AI Agent Organization Suite
**Mode**: Ultra-Strict (Autonomous-Safe)

## Protocol Purpose

This protocol ensures deterministic session rehydration, prevents context drift, and enables predictable agent behavior across sessions and repositories.

**Target**: Session rehydration in <30 seconds with zero context guessing.

## 1. Session Outcome Digest

### Definition

A **digest** is a derived, machine-readable summary of session outcomes. It is NOT a narrative document.

### File Location

```
~/.claude/context/projects/{PROJECT_KEY}/sessions/{SESSION_ID}.digest.yaml
```

### Schema (v1.0)

```yaml
# Session Outcome Digest
# Generated by: CTX-00 (single writer)
# Derived from: summary.md, events.ndjson, accepted HOFFs

session_id: ai-agents-org_2025-12-27_1430
project_key: ai-agents-org
created: 2025-12-27T16:30:00Z

# What changed (IDs only, no prose)
tasks_completed:
  - TASK-017
  - TASK-018

tasks_blocked:
  - TASK-019: "Awaiting API-GOV review"

decisions_made:
  - DEC-009

patterns_created:
  - PATTERN-011

lessons_created: []

handoffs_created:
  - HOFF-043

escalations_created: []

# Pointers to artifacts
artifacts:
  summary: ./ai-agents-org_2025-12-27_1430_summary.md
  events: ./events.ndjson

# Lineage
continuity:
  continues_from: ai-agents-org_2025-12-26_0930
  branched_from: null
```

### Generation Rules

| Rule | Enforcement |
|------|-------------|
| Single writer | CTX-00 only |
| Max lines | 12 content lines (excluding comments/whitespace) |
| No prose | IDs and paths only |
| Derived only | Must be computable from source files |
| Immutable | Once written, never modified |

### Validation

| Field | Validation |
|-------|------------|
| `session_id` | Must match filename |
| `tasks_*` | All must exist in `active_tasks.json` or `events.ndjson` |
| `decisions_made` | All must exist in `shared/decisions/` |
| `patterns_created` | All must exist in `shared/patterns/` |
| `lessons_created` | All must exist in `shared/lessons/` |
| `handoffs_created` | All must exist in `handoffs/` |
| `artifacts.*` | All paths must exist |

**Fail-Closed**: Digest with invalid references is rejected. CTX-00 must repair.

## 2. Context Confidence Signals

### Definition

Confidence signals are computed boolean flags indicating session readiness. They are NOT opinions.

### Location

Added to `landing.yaml` under `confidence:` key.

### Schema

```yaml
confidence:
  context_complete: true      # All required files exist
  handoffs_resolved: false    # No PENDING HOFFs for this project
  blockers_known: true        # All BLOCKED tasks have documented reason
  escalation_open: false      # No OPEN ESCs for this project
  digest_current: true        # Latest session has digest
```

### Computation Rules

| Flag | True when |
|------|-----------|
| `context_complete` | `landing.yaml` exists AND all `pointers.*` resolve |
| `handoffs_resolved` | Zero HOFFs with `status: PENDING` for project |
| `blockers_known` | All tasks with `state: BLOCKED` have `blocker_reason` field |
| `escalation_open` | At least one ESC with `status: OPEN` for project |
| `digest_current` | Digest exists for most recent session |

### Confidence Thresholds

| Threshold | Required Flags | Agent Behavior |
|-----------|----------------|----------------|
| **GREEN** | All true except `escalation_open` | Autonomous continuation allowed |
| **YELLOW** | `context_complete` + `blockers_known` true | Proceed with caution, log warning |
| **RED** | `context_complete` false OR `blockers_known` false | HALT. Request CTX-00 repair. |

### Enforcement

```
if confidence.context_complete == false:
  HALT("Context incomplete. CTX-00 must repair landing.yaml.")

if confidence.handoffs_resolved == false:
  WARN("Unresolved handoffs exist. Check before proceeding.")
  # Continue only if explicit user override

if confidence.escalation_open == true:
  WARN("Open escalation. Human decision may be required.")
  # Continue with awareness
```

**ORC-00 must refuse autonomous continuation when threshold is RED.**

## 3. Read Order Contract

### Definition

All agents MUST read context files in this exact order before executing any task. No exceptions.

### Mandatory Read Order

```
1. ~/.claude/CLAUDE.md                                    # Global constraints
2. ~/.claude/context/projects/{PROJECT_KEY}/landing.yaml  # Project truth
3. ~/.claude/context/projects/{PROJECT_KEY}/sessions/{LATEST_SESSION_ID}.digest.yaml
4. ~/.claude/context/handoffs/HOFF-*_{PROJECT_KEY}_*.yaml WHERE status=PENDING
5. ~/.claude/context/escalations/ESC-*_{PROJECT_KEY}_*.md WHERE status=OPEN
6. Run artifacts (active_tasks.json, events.ndjson, etc.)
```

### Enforcement Rules

| Rule | Consequence |
|------|-------------|
| Skip step 1-2 | INVALID session. Agent must restart join sequence. |
| Skip step 3 | WARNING. Log missing digest, proceed with full summary read. |
| Skip step 4-5 | Silent handoff drops. HOFF sender must escalate. |
| Read order violation detected | Log to `events.ndjson`, flag for review. |

### Detection

Agents MUST emit a `context.loaded` event after completing read order:

```json
{
  "event": "context.loaded",
  "timestamp": "2025-12-27T14:30:05Z",
  "agent": "api-core",
  "session_id": "ai-agents-org_2025-12-27_1430",
  "read_order_complete": true,
  "files_read": [
    "CLAUDE.md",
    "landing.yaml",
    "ai-agents-org_2025-12-27_1430.digest.yaml",
    "HOFF-043_api-core_test-00.yaml"
  ],
  "confidence_threshold": "GREEN"
}
```

### Invariant Statement (for CLAUDE.md)

> **Read Order Invariant**: Before executing any task, agents MUST complete the read order contract defined in SESSION_CONTINUITY_PROTOCOL.md. Agents that skip steps 1-2 are in an invalid state. Agents that skip steps 4-5 risk silent handoff drops. Violation of read order is a loggable event.

## 4. Session Continuity Tags

### Definition

Continuity tags establish session lineage. They enable branching, parallel experiments, and safe abandonment.

### Schema

Added to: `landing.yaml`, `HOFF`, `summary.md`, `digest.yaml`

```yaml
continuity:
  continues_from: SESSION_ID | null   # Linear continuation
  branched_from: SESSION_ID | null    # Parallel experiment
  abandoned: false                     # Explicitly abandoned
  abandon_reason: null                 # Required if abandoned=true
```

### Continuity Types

| Type | `continues_from` | `branched_from` | Meaning |
|------|------------------|-----------------|---------|
| Fresh start | null | null | New project or clean slate |
| Linear | SESSION_ID | null | Direct continuation |
| Branch | null | SESSION_ID | Parallel experiment |
| Merge | SESSION_ID | SESSION_ID | Converging branches |

### Rules

| Rule | Enforcement |
|------|-------------|
| Single parent | `continues_from` and `branched_from` cannot both be non-null unless merging |
| Valid references | All SESSION_IDs must exist in `sessions/` |
| Abandon requires reason | If `abandoned: true`, `abandon_reason` is required |
| Immutable lineage | Once set, `continues_from` and `branched_from` cannot change |

### Abandonment Protocol

When a session is abandoned:

1. Set `abandoned: true` in digest
2. Provide `abandon_reason` (max 100 chars)
3. Update `landing.yaml` to point to parent session
4. Do NOT delete abandoned session files (audit trail)

```yaml
continuity:
  continues_from: ai-agents-org_2025-12-26_0930
  branched_from: null
  abandoned: true
  abandon_reason: "Approach deprecated. Superseded by TASK-020."
```

## 5. CTX-00 Responsibilities

CTX-00 is the **state compiler** for session continuity.

### At Session Start

1. Verify `landing.yaml` exists (cold start if not)
2. Compute confidence signals
3. If confidence threshold is RED, halt and report
4. Load latest digest
5. Verify read order contract can be satisfied

### At Session End

1. Generate `{SESSION_ID}.digest.yaml`
2. Update `landing.yaml` with new session pointer
3. Update confidence signals
4. Set continuity tags

### On Digest Generation Failure

1. Log failure to `events.ndjson`
2. Set `confidence.digest_current: false`
3. Create ESC if failure persists across 2 sessions

## 6. ORC-00 Responsibilities

ORC-00 enforces the read order contract and confidence thresholds.

### Before Task Dispatch

1. Verify agent completed read order (check `context.loaded` event)
2. Check confidence threshold
3. If RED, refuse dispatch
4. If YELLOW, log warning, proceed if user override

### On Confidence Violation

```
if agent_attempts_task AND confidence == RED:
  REJECT("Confidence threshold not met. CTX-00 must repair.")
  emit task.blocked with reason="confidence_threshold_red"
```

## 7. Prohibited Actions

These actions are explicitly forbidden under this protocol:

| Forbidden | Why |
|-----------|-----|
| Agents writing to `landing.yaml` | Single-writer violation |
| Agents creating digests | CTX-00 only |
| Skipping read order steps 1-2 | Invalid session state |
| Modifying lineage after creation | Immutable audit trail |
| Free-form notes in digests | Machine-readable only |
| Per-agent memory files | Prevents split-brain |
| Narrative history in Hive | Not an archive |

## Quick Reference

### Rehydration Checklist (<30 seconds)

1. Read `CLAUDE.md` (cached)
2. Read `landing.yaml` (project state)
3. Check confidence signals
4. Read latest digest (12 lines max)
5. Scan PENDING HOFFs (typically 0-2)
6. Scan OPEN ESCs (typically 0-1)
7. Ready to execute

### Confidence Quick Check

```
GREEN  = All good, proceed autonomously
YELLOW = Caution, log and proceed
RED    = HALT, repair required
```

### Digest Quick Validation

```
Valid digest:
  - Exists at correct path
  - All IDs resolve
  - Max 12 content lines
  - No prose, only IDs and paths
```
